Fundamentos del Desarrollo Frontend

Introducción a HTML y HTML5

El lenguaje de marcado de hipertexto (HTML) es el estándar fundamental para la creación de páginas web. HTML5, la versión más reciente, no solo define el contenido, sino que introdujo capacidades multimedia avanzadas y elementos semánticos que mejoran la accesibilidad y el SEO.

Estructura y Seccionado

Una página HTML5 moderna se organiza mediante etiquetas semánticas que indican la función de cada sección:

  • <header>: Contiene el logotipo y la navegación principal.
  • <main>: Define el contenido central y único del documento.
  • <section>: Agrupa contenido relacionado temáticamente.
  • <article>: Representa contenido independiente como un post o noticia.
  • <aside>: Contenido lateral o tangencial.
  • <footer>: Información de cierre y créditos.
Ejemplo de Estructura HTML5:
<!DOCTYPE html>
<html lang="es">
<head>
    <title>Mi Proyecto UAA</title>
</head>
<body>
    <header>
        <nav>Menú de Navegación</nav>
    </header>
    <main>
        <section>
            <h1>Contenido Académico</h1>
            <p>Información técnica aquí.</p>
        </section>
    </main>
</body>
</html>

Cascading Style Sheets (CSS)

CSS es el lenguaje encargado de la capa visual. Permite separar el contenido (HTML) del diseño, facilitando el mantenimiento y la consistencia en todo el sitio.

Propiedades más utilizadas:
  • Color y Background: color, background-color y linear-gradient.
  • Modelo de Caja: margin, padding, border y box-shadow.
  • Tipografía: font-family, font-size y line-height.
  • Layout: display: flex (Flexbox) y display: grid (Grid Layout).
  • Posicionamiento: position: sticky y fixed.

El Document Object Model (DOM)

El DOM es una interfaz de programación que representa un documento HTML o XML como una estructura de árbol de nodos. Cada etiqueta, atributo y fragmento de texto es un objeto que puede ser manipulado programáticamente.

Funcionamiento y Utilidad:

Cuando el navegador carga una página, crea este modelo en memoria. Gracias al DOM, lenguajes como JavaScript pueden:

  • Cambiar elementos HTML dinámicamente.
  • Modificar atributos de estilo CSS en tiempo real.
  • Reaccionar a eventos (clics, envíos de formularios, scroll).
  • Añadir o eliminar nodos del documento sin recargar la página.

"El DOM es el puente que convierte una página estática en una aplicación web interactiva."

Tecnologías de Web APIs

Las APIs (Interfaces de Programación de Aplicaciones) son mecanismos fundamentales que permiten a dos componentes de software comunicarse entre sí mediante un conjunto de definiciones y protocolos.

Actúan como una capa intermedia que procesa las transferencias de datos entre sistemas, permitiendo a las empresas abrir sus datos y funcionalidades a desarrolladores externos, socios comerciales y departamentos internos.

Google Maps

PayPal

WhatsApp

Ciclo de Vida del Desarrollo de una API

Implica identificar los requisitos funcionales y no funcionales, casos de uso y datos a manipular. Es fundamental definir los objetivos antes de comenzar el desarrollo.

Se definen los endpoints (URLs específicas) y los métodos HTTP asociados (GET, POST, PUT, DELETE). Por ejemplo, en un blog, un endpoint sería /posts para obtener la lista de entradas.

Se utilizan frameworks como Express.js o Flask. La seguridad es crucial, implementando autenticación y autorización para proteger la API de accesos no autorizados.

Microservicios: El Futuro de la Escalabilidad

La arquitectura de microservicios permite a los equipos desarrollar y desplegar aplicaciones mediante servicios aislados, granulares y modulares. Gigantes tecnológicos han establecido los estándares para este paradigma:

IBM

IBM destaca que las APIs actúan como capas intermedias esenciales para el intercambio de datos corporativos. Su enfoque en microservicios se centra en entornos híbridos y empresariales, permitiendo que departamentos internos y socios comerciales interactúen de forma segura mediante reglas definidas.

AWS

Amazon define las APIs como mecanismos que permiten la comunicación entre software mediante protocolos estrictos. AWS es el líder mundial en infraestructura cloud, ofreciendo servicios como Lambda para computación "serverless", lo que permite ejecutar microservicios sin gestionar servidores físicos.

INTEL

Mientras que el software define el servicio, Intel proporciona la base de hardware necesaria. Sus procesadores Xeon están optimizados para la virtualización y el manejo de contenedores, asegurando que las latencias de comunicación entre microservicios se mantengan al mínimo en centros de datos masivos.

Contenedores y Orquestación

Definición y Ventajas

Un contenedor es una unidad ligera de software que empaqueta una aplicación junto con todas sus dependencias (bibliotecas, configuraciones y archivos necesarios) para que pueda ejecutarse de manera consistente en cualquier entorno.

Arranque rápido
Aislamiento total
Bajo consumo
Escalabilidad

Líderes y Ecosistemas

Google y Kubernetes

Google es el creador original de Kubernetes (K8s), el sistema de orquestación líder que administra múltiples contenedores a gran escala de forma automática.

AWS (Amazon Web Services)

Amazon ofrece servicios como ECS (Elastic Container Service) y EKS, permitiendo ejecutar contenedores en la nube de manera masiva para infraestructuras empresariales distribuidas.

Arquitectura de Docker

Docker es una plataforma para crear y ejecutar contenedores individuales. Su arquitectura se basa en:

  • Docker Engine: El motor que gestiona los contenedores.
  • Imágenes: Plantillas de solo lectura para crear contenedores.
  • Dockerfile: Archivo de configuración con las instrucciones de empaquetado.

Arquitectura de Kubernetes

Kubernetes coordina grupos de contenedores (Pods) mediante una arquitectura de clúster:

  • Control Plane: Gestiona el estado deseado del clúster.
  • Nodos: Máquinas (físicas o virtuales) donde corren los contenedores.
  • Kubelet: Agente que asegura que los contenedores estén sanos en cada nodo.

Protocolo de Seguridad Integral (JWT)

JSON Web Token (JWT) es un estándar de autenticación definido en la RFC 7519 que permite transmitir información de forma segura entre dos partes mediante un objeto JSON firmado digitalmente.

HEADER
Algoritmo (ej. HS256)
PAYLOAD
Datos del usuario (Claims)
SIGNATURE
Verificación de integridad
Proceso de Autenticación:
  1. El usuario ingresa sus credenciales.
  2. El servidor valida y genera el JWT.
  3. El cliente guarda el token y lo envía en el encabezado de cada solicitud posterior: Authorization: Bearer TOKEN.
  4. El servidor verifica la firma y concede el acceso si es válido.
